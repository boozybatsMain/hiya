# Meta Pixel и Events Manager

Пиксель Meta установлен на лендинге и принадлежит бизнес-портфолио «Hiya App».
Клиентская часть (как события шлются с сайта) — в [`tracking.md`](tracking.md).
Правила работы с рекламным кабинетом (что можно без вопроса, что требует
подтверждения) — в `CLAUDE.md`.

## Какие события шлёт сайт

| Событие | Тип | Когда |
|---|---|---|
| `PageView` | стандартное | загрузка страницы |
| `OpenSignup {source}` | trackCustom | открытие модалки заявки |
| `Lead {method}` + `eventID` | стандартное | подтверждённая отправка заявки |

`eventID` зарезервирован под дедупликацию с серверным **Conversions API**
(план: Firebase Function как лид-эндпоинт). Пока CAPI не подключён — пиксель
работает в одиночку, этого достаточно для этапа вейтлиста.

**Automatic Advanced Matching включён** (Events Manager → Настройки): пиксель
хэширует и передаёт email/имя/телефон из форм — повышает матчинг конверсий
с аккаунтами FB/IG.

Пиксель гейтится общим флагом `live`: свой трафик (`?mytraffic=off`) и localhost
события не шлют.

## Где смотреть события

Events Manager → **Наборы данных** → hiya:
- **Обзор** — графики событий (задержка отображения до ~30 минут; проверь
  диапазон дат — «Сегодня» не всегда включён по умолчанию).
- **Тестирование событий** — реалтайм-отладка: открой сайт с телефона и смотри,
  как прилетают PageView/OpenSignup/Lead.
- **Диагностика** — предупреждения о качестве данных.

## Зачем это кабинету

1. **Атрибуция** — в Ads Manager видно, какая кампания/креатив принёс заявку
   (колонки «Лиды» / кастомные конверсии).
2. **Оптимизация** — цель «Лиды» с оптимизацией на `OpenSignup` (пока лидов
   мало) и затем на `Lead`. Алгоритму нужно ~50 событий оптимизации в неделю,
   чтобы выйти из фазы обучения.
3. **Аудитории** — ретаргетинг («открыл модалку без заявки»), lookalike от
   лидов (имеет смысл от ~100+ лидов).

## Известные грабли

- **Блокировщики рекламы** режут `connect.facebook.net/…/fbevents.js`
  (в браузере видно как 503). В in-app браузере Instagram — основном источнике
  трафика — блокировщиков не бывает, покрытие там нормальное. Не паниковать,
  если в своём десктопном браузере пиксель «не работает».
- **Ошибка #2446880 «Требуется номер WhatsApp»** может всплывать при
  редактировании объявления (creative-01-dark-phone): это отвязанный WhatsApp
  у страницы «Хэйя», к настройкам объявления отношения не имеет и показы не
  блокирует, пока черновик с ошибкой не публикуется. Лечится привязкой номера
  WhatsApp к странице либо сбросом авто-черновика.
- **«Ограниченные затраты на исключённые места размещения»** в адсете включены
  (дефолт Meta): до ~5% бюджета может уходить на выключенные плейсменты —
  поэтому в GA встречается `utm_source=fb` при IG-only настройке.
- **Изменения адсета** (гео, плейсменты, событие оптимизации, креатив, бюджет
  >±20%) перезапускают фазу обучения. Менять пакетом, одной публикацией,
  не чаще ~раза в неделю.
- Страницы «Хэйя» (FB) и hiyawrld (IG) в портфолио **не добавлены** — нужен
  полный доступ владельца страницы (Business Settings → Аккаунты → Страницы).
  На работу пикселя и рекламы не влияет.
